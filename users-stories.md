# User Stories – Smart Clinic Management System

This document defines the user stories for the three primary roles in the Smart Clinic Management System: **Admin**, **Patient**, and **Doctor**. Each user story is written using the standard format:

> **As a [role], I want to [goal], so that I can [benefit].**

---

# 1. Admin User Stories

### **1.1 Manage User Accounts**
- As an admin, I want to create new doctor and patient accounts, so that they can access the system.
- As an admin, I want to update user details, so that account information stays accurate.
- As an admin, I want to deactivate or delete accounts, so that I can remove inactive or unauthorized users.

### **1.2 Role & Access Control**
- As an admin, I want to assign roles (admin/doctor/patient), so that users receive the correct permissions.
- As an admin, I want to restrict access to certain pages or actions, so that the system remains secure.

### **1.3 Appointment Monitoring**
- As an admin, I want to view all appointment data, so that I can monitor clinic operations.
- As an admin, I want to cancel or reschedule appointments if needed, so that I can assist with manual conflicts.

### **1.4 System Settings**
- As an admin, I want to update system-wide settings (clinic hours, departments, etc.), so that the platform remains accurate.
- As an admin, I want to view system usage analytics, so that I can track performance and optimize operations.

---

# 2. Patient User Stories

### **2.1 Account & Profile**
- As a patient, I want to create an account, so that I can access the clinic services.
- As a patient, I want to update my personal details, so that my medical information stays current.

### **2.2 Appointment Scheduling**
- As a patient, I want to view available doctors and their schedules, so that I can choose a suitable appointment time.
- As a patient, I want to book an appointment, so that I can receive medical consultation.
- As a patient, I want to cancel or reschedule an appointment, so that I can adjust plans conveniently.

### **2.3 Medical Records**
- As a patient, I want to view my prescription history, so that I can keep track of my treatments.
- As a patient, I want to download or view my medical reports, so that I can store them for personal use.

### **2.4 Notifications**
- As a patient, I want to receive appointment reminders, so that I don’t miss my scheduled visits.

---

# 3. Doctor User Stories

### **3.1 Account & Schedule Management**
- As a doctor, I want to manage my profile information, so that patients see accurate details.
- As a doctor, I want to set my available days and hours, so that patients can book appointments appropriately.

### **3.2 Appointment Management**
- As a doctor, I want to view all my upcoming appointments, so that I can prepare for consultations.
- As a doctor, I want to mark appointments as completed, so that the system updates the appointment status.
- As a doctor, I want to cancel appointments when necessary, so that I can adjust my schedule.

### **3.3 Patient Medical Records**
- As a doctor, I want to view patient histories before consultations, so that I can provide informed diagnoses.
- As a doctor, I want to upload or update prescriptions, so that patients receive correct medications.
- As a doctor, I want to add notes to patient records, so that future consultations have context.

### **3.4 Notifications**
- As a doctor, I want to receive notifications for newly booked appointments, so that I’m aware of upcoming patients.
